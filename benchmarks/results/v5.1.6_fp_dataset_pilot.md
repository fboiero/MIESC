# Labeled FP Dataset — Pilot (v5.1.6)

**Date**: 2026-04-12
**Goal**: Bootstrap a seed labeled dataset so `AuditorTrainedFPClassifier`
(shipped in v5.1.5, dormant since) has something real to train on.

---

## 1. Method

`scripts/bootstrap_fp_dataset.py` auto-labels findings produced by
`miesc scan` on two corpora using three heuristic rules:

1. **Noise types** → `label=False` (FP). Covers Aderyn / Slither info-
   level detectors: `solc-version`, `naming-convention`, `useless-public-
   function`, `push-zero-opcode`, `pragma-not-pinned`, etc.
2. **High/Critical finding whose type contains keywords matching the
   fixture's ground-truth vulnerability class** → `label=True` (TP).
3. **Info severity on a type that does NOT match the fixture's GT class**
   → `label=False` (off-class info noise).

Everything else is skipped (returns `None`) to keep the seed clean.

## 2. Result

Ran on the 11 Rekt exploits with fixtures + the SmartBugs-curated corpus
available locally (28 contracts across 5 categories):

| Source     | Samples produced |
|-----------|-------:|
| Rekt      | 24     |
| SmartBugs | 98     |
| **Total** | **122** (3 TP / 119 FP) |

Dataset is highly imbalanced: the auto-labeling is conservative by design
(reject "medium severity on unrelated type" as ambiguous), so most
surviving labels are the obvious FPs from info-level noise detectors.

### Breakdown by heuristic reason

```
   78  noise-type:useless-public-function
   36  noise-type:unspecific-solidity-pragma
    4  noise-type:constants-instead-of-literals
    3  gt-match:reentrancy
    1  noise-type:boolean-equality
```

## 3. Classifier training on the seed

Ran `AuditorTrainedFPClassifier.train()` on the seed JSONL:

```
train_samples: 97
test_samples:  25
accuracy:      1.00
precision:     1.00
recall:        1.00
f1:            1.00
```

**These metrics are not meaningful yet.** With 3 TPs in 122 samples, the
train/test split's stratification leaves roughly 0.6 TPs in the test fold;
any classifier that predicts FP for everything achieves ≥97 % accuracy,
and the precision/recall metrics on the TP class become degenerate.

What the 100 % does validate:

- The training pipeline runs end-to-end (feature extraction, split,
  sklearn fit, metric computation, model persistence).
- The model writes to `~/.miesc/models/fp_auditor_classifier.pkl`
  successfully.
- `predict_fp_probability()` returns a float in $[0,1]$ on subsequent calls.

## 4. Why the seed is imbalanced (and how to fix it)

Two structural reasons:

1. **Exploit-side fixtures**: The Rekt corpus we have locally consists of
   attacker-side exploit harnesses, not the victim-side vulnerable
   protocols. Slither/Aderyn find very little on these files (typically
   one or two critical findings and some info noise). For Euler the scan
   yields 3 findings total (1 HIGH, 2 LOW).
2. **Heuristic is deliberately conservative**: we only label when both
   severity and type align. Medium-severity findings with ambiguous types
   are dropped, which removes the bulk of real signal.

The honest fix is a **labeled victim-side corpus**: re-run MIESC against
the ORIGINAL vulnerable contracts (Euler's EToken code, Ronin Bridge
validator, Beanstalk's governance), not the attacker exploits. That
corpus will emit many more HIGH/CRITICAL findings that can be ground-
truth labeled against the public post-mortems.

## 5. What this pilot validates

- End-to-end training flow is live: scan → label → JSONL → train → pickle.
- `AuditorTrainedFPClassifier` is no longer dormant — it has a concrete
  (if weak) seed model on disk.
- The bootstrap script is reproducible: re-running with more data will
  continue to improve the model.

## 6. What this pilot does NOT validate

- Classifier precision/recall on real audit workloads (needs a balanced
  corpus; the 100 % numbers above are degenerate, not meaningful).
- The classifier's ability to distinguish subtle FPs from subtle TPs
  (needs auditor-labeled data with intentional hard cases).

**Next steps**:

1. Assemble a victim-side exploit corpus (~30 contracts), re-run the
   bootstrap, and aim for ≥30 % TP ratio.
2. Complement auto-labels with 50+ hand-labeled findings on contracts
   the user has already audited.
3. Re-train and measure on a held-out test set of real auditor verdicts.

---

**Reproduce**:
```
python3 scripts/bootstrap_fp_dataset.py --train
# Skip one corpus:
python3 scripts/bootstrap_fp_dataset.py --skip-smartbugs --train
# Custom output path:
python3 scripts/bootstrap_fp_dataset.py --output /tmp/my_labels.jsonl
```
