üåü General Questions

What is MIESC Agent Protocol? ‚ñº

MIESC Agent Protocol is an open standard that enables any security tool to integrate with any platform instantly. Think of it as "USB for security tools" - just as USB created a universal hardware interface, MIESC creates a universal software interface for security analysis.

The protocol defines a standard interface (SecurityAgent) that security tools implement, allowing them to be discovered, selected, and executed through a unified orchestration system.

Is MIESC open source? ‚ñº

Yes, 100%! MIESC Agent Protocol is open source under the MIT license. This means:

  • Anyone can use it for free
  • You can modify and customize it
  • You can build commercial products on top of it
  • No vendor lock-in or proprietary restrictions
  • Community-driven development

The code is publicly available on GitHub.

Who controls MIESC? ‚ñº

The community. MIESC is not controlled by a single company. It's governed by its users through:

  • Open source collaboration on GitHub
  • Community discussions and RFCs
  • Democratic decision-making on protocol changes
  • Transparent development process

Similar to successful open standards like HTTP, USB, or JSON, MIESC is designed to be neutral and vendor-independent.

How does MIESC make money? ‚ñº

MIESC Agent Protocol doesn't make money - it's a standard, not a product.

Think of it like USB or HTTP - they're standards that enable ecosystems. The value comes from:

  • Tool developers reaching more users
  • Auditors saving time with unified workflows
  • Protocols getting better security coverage
  • The entire ecosystem growing together

Individual agents can charge for their services (by setting a cost parameter), but the protocol itself is free forever.

Why should I care about another "standard"? ‚ñº

Great question! Here's why MIESC is different:

  1. Solves Real Pain: The fragmentation of 50+ security tools is costing teams weeks of integration work
  2. Network Effects: More agents = more value for everyone (not zero-sum)
  3. Easy to Join: Implement the interface in ~2 hours, get instant distribution
  4. Actually Open: MIT licensed, community-governed, no gatekeepers
  5. Already Working: Not vaporware - running code with multiple agents

If you work with smart contract security, this will either save you time (as a user) or give you distribution (as a tool developer).

üîß For Tool Developers

How do I integrate my security tool? ‚ñº

Integration typically takes 1-2 hours. Here's the process:

  1. Implement the Interface: Create a Python class that inherits from SecurityAgent
  2. Define Metadata: Specify name, version, capabilities, supported languages
  3. Implement analyze(): Call your tool and convert output to standard format
  4. Test Locally: Copy to ~/.miesc/agents/ and test discovery
  5. Publish: Submit to marketplace or distribute directly

See the Developer Guide for a complete tutorial and example agent for reference code.

Do I need to rewrite my tool in Python? ‚ñº

No! Your tool can be written in any language. The agent is just a thin wrapper.

For example, our Slither agent simply calls the Slither CLI and parses its output. The Semgrep example does the same. Your agent can:

  • Call external CLI tools via subprocess
  • Make HTTP requests to your API
  • Load native libraries via ctypes/cffi
  • Use Docker containers

The Python code is just for integration, not for rewriting your core analysis logic.

Can I charge for my agent? ‚ñº

Yes! Set the cost property to your price per analysis: 

@property
def cost(self) -> float:
    return 5.00  # $5 per analysis

MIESC handles payment processing (in future versions). Users can filter by free_only=True if they only want free agents.

What if my tool has large dependencies? ‚ñº

That's fine! There are several approaches:

  • Optional dependencies: Document them in your agent description
  • Check in is_available(): Return False if dependencies aren't installed
  • Docker container: Package your tool and dependencies in a container
  • Cloud API: Host your tool remotely and call it via HTTP

Users who want your agent will install the dependencies. The protocol supports this gracefully.

How do I get users? ‚ñº

Once published, your agent is automatically discoverable by:

  • MIESC CLI users (miesc agents list)
  • Agent marketplace (coming soon)
  • IDE plugins (VS Code, etc.)
  • CI/CD integrations
  • AI assistants via MCP

Every MIESC user becomes a potential user of your agent. No custom integrations needed.

üîç For Security Auditors

How is this different from running tools manually? ‚ñº

Time savings and consistency:

Task Manual With MIESC
Install 5 tools 2.5 hours 5 minutes
Run tools 25 minutes (sequential) 5 minutes (parallel)
Consolidate results 30 minutes Automatic
Total ~3.5 hours ~10 minutes

Plus: Unified dashboard, no context switching, reproducible configs.

Can I still use my favorite tools? ‚ñº

Absolutely! MIESC doesn't replace your tools - it orchestrates them.

You can:

  • Select specific agents: SelectionCriteria(specific_agents=['slither', 'mythril'])
  • Run tools independently if needed
  • Use MIESC for some projects and manual workflows for others
  • Gradually adopt as agents for your tools become available
How accurate are the results? ‚ñº

MIESC's accuracy = the accuracy of the underlying tools. We don't modify analysis results.

What MIESC adds:

  • Deduplication: Remove duplicate findings across tools
  • Aggregation: Combine related findings
  • Prioritization: Rank by severity and confidence
  • Context: Show findings from multiple perspectives

Defense-in-depth: Running multiple tools catches more issues than any single tool alone.

Do I need to learn Python? ‚ñº

No. As a user, you interact via:

  • CLI: miesc analyze contract.sol --auto
  • Web UI: Upload contracts, click analyze
  • IDE: Right-click ‚Üí "Analyze with MIESC"
  • AI Assistant: "Claude, analyze this contract"

Python is only needed if you want to build custom agents or integrations.

What languages are supported? ‚ñº

Currently supported:

  • Solidity
  • Vyper
  • Rust (CosmWasm)

Coming soon:

  • Cairo (StarkNet)
  • Move (Aptos/Sui)
  • Yul
  • Fe

The protocol is language-agnostic - agents declare which languages they support via supported_languages property.

How does agent discovery work? ‚ñº

The AgentRegistry discovers agents from multiple sources:

  1. Built-in agents: Shipped with MIESC (src/agents/)
  2. User agents: Installed locally (~/.miesc/agents/)
  3. Marketplace agents: Downloaded from public registry (future)

Discovery is automatic at runtime using Python's importlib. Any Python file with a SecurityAgent subclass is detected and registered.

Can agents communicate with each other? ‚ñº

Yes! Agent communication is fully implemented in v3.3.0 via Model Context Protocol (MCP).

The "agent composition" feature allows:

  • Agents passing results to other agents
  • Pre-processing agents (normalize code)
  • Analysis agents (find vulnerabilities)
  • Post-processing agents (prioritize/dedupe)
  • Complex analysis pipelines
How do you handle security of agents? ‚ñº

Current approach:

  • Built-in agents are vetted by MIESC maintainers
  • User-installed agents run with user's permissions
  • Users are responsible for vetting agents they install

Future enhancements (v3.0):

  • Sandboxed execution (containers, VMs)
  • Permission system (what resources agents can access)
  • Code signing and verification
  • Community trust ratings
  • On-chain verification registry
What's the performance overhead? ‚ñº

Minimal. The orchestrator adds:

  • Agent discovery: ~50-100ms (one-time at startup)
  • Agent selection: ~10ms per contract
  • Result consolidation: ~50ms per 100 findings

Benefits from parallelization: Running agents in parallel typically reduces total time by 40-60% compared to sequential execution.

Example: 3 agents (10s, 15s, 5s) = 15s parallel vs 30s sequential ‚Üí 50% faster!
